# Сценарий защиты — Лабы 2 и 3

## ЛАБА 2 — Razor Pages

### Шаг 1: Показываешь браузер
> "Вот приложение на Razor Pages, работает на localhost. Здесь форма для расчёта скидки на товар."

Вводишь имя, цену, нажимаешь кнопку — показываешь что результат появился.

### Шаг 2: Осипов спросит "Как формируется адрес в браузере?"
> "Адрес формируется через **систему маршрутизации — Routing**. В ASP.NET Razor Pages маршрут определяется **именем файла** в папке Pages. Файл `Pages/Product.cshtml` автоматически доступен по адресу `/Product`."

**Сразу открываешь `Program.cs`**, показываешь строку 21:
> "Вот, `app.MapRazorPages()` — эта строка регистрирует все Razor Pages в системе маршрутизации."

### Шаг 3: "Как формируется картинка / страница?"
> "Есть мастер-страница `_Layout.cshtml` — это общий шаблон. В ней есть `@RenderBody()` — сюда подставляется содержимое конкретной страницы. То есть Layout — это обёртка, а каждая страница — её содержимое."

Открываешь `_Layout.cshtml`, показываешь `@RenderBody()`.

### Шаг 4: "Покажите обработчик"
Открываешь `Product.cshtml.cs`:
> "Вот `OnPost` — срабатывает при отправке формы, считает скидку 18%. А `OnPostDiscont` — принимает произвольный процент скидки от пользователя. Razor Pages использует соглашение: `OnGet` для GET-запросов, `OnPost` для POST."

### Шаг 5: Если спросит про ссылки
> "Ссылки формируются через **Tag Helpers**: `asp-page="/Product"` — это серверный атрибут, ASP.NET сам генерирует правильный href."

## ЛАБА 3 — MVC

### Шаг 1: Показываешь браузер
> "Это MVC-приложение для управления кредитами. На главной — таблица кредитных продуктов."

Показываешь таблицу, потом нажимаешь "Подать заявку".

### Шаг 2: Осипов спросит "Как формируется адрес?"
> "Здесь тоже **система маршрутизации — Routing**, но работает по-другому. В MVC маршрут задаётся **шаблоном**: `{controller}/{action}/{id?}`."

**Открываешь `Program.cs`**, строка 49-51:
> "Вот: `app.MapControllerRoute`, pattern — `{controller=Home}/{action=Index}/{id?}`. То есть адрес `/Home/CreateBid` — это контроллер `Home`, метод `CreateBid`."

### Шаг 3: "Из чего состоит контроллер?"
Открываешь `HomeController.cs`:
> "Контроллер состоит из: **конструктор** — сюда через DI приходят зависимости (логгер, контекст БД). **Action-методы** — `Index()`, `CreateBid()`. Каждый метод возвращает `View()` — ASP.NET ищет представление по пути `Views/Home/Index.cshtml`."

### Шаг 4: "Что за Authorize?"
> "Атрибут `[Authorize]` на методе `CreateBid` означает, что без авторизации пользователь не попадёт на эту страницу — его перенаправит на логин. Используется **ASP.NET Core Identity** — библиотека для регистрации, входа и ролей."

Можешь показать: выйти из аккаунта -> нажать "Подать заявку" -> редирект на логин.

### Шаг 5: "AJAX?"
Открываешь `CreateBid.cshtml`, скроллишь вниз к `<script>`:
> "Поиск заявок работает через AJAX — `fetch('/Home/BidSearch?creditHead=...')`. Страница **не перезагружается**, результат вставляется в div. На сервере метод `BidSearch` возвращает **частичное представление** `_BidSearchPartial`."

### Шаг 6: "Разница с Razor Pages?"
> "В Razor Pages **файл = endpoint**: `Product.cshtml` сам обрабатывает запросы через `OnGet/OnPost`. В MVC — **разделение**: контроллер обрабатывает логику, а view только отображает. MVC даёт больше контроля, Razor Pages проще для страничных сценариев."

## Ключевые фразы — выучи наизусть

| Когда | Говори |
|-------|--------|
| Адрес? | "Система маршрутизации, Routing" |
| Razor Pages маршрут? | "Имя файла в папке Pages = адрес" |
| MVC маршрут? | "Шаблон controller/action/id" |
| Страница собирается? | "`_Layout.cshtml`, `@RenderBody()`" |
| Ссылки? | "Tag Helpers: `asp-page`, `asp-controller`, `asp-action`" |
| Authorize? | "Identity, ограничение доступа, роли admin/user" |

## Главное правило

Осипов любит когда ты **сам показываешь код**, не дожидаясь наводки. Сказал "Routing" — сразу переключайся на `Program.cs` и показывай строку. Не жди пока он скажет "а покажите".
