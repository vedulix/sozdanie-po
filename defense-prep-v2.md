### "Как формируется адрес в браузере? Система маршрутизации."

**Для Razor Pages (Лаба 2):**
- В `Program.cs` → `app.MapRazorPages()` — регистрирует все страницы
- Адрес = **имя файла** в папке Pages: `Pages/Product.cshtml` → `/Product`
- `Pages/Index.cshtml` → `/` (корневой)
- Параметры: `?name=Tomato&price=31` — query string
- Ссылки между страницами: `<a asp-page="/Product">Продукт</a>` — tag helper формирует href
- Это называется **система маршрутизации (Routing)**

**Для MVC (Лаба 3):**
- В `Program.cs` → `app.MapControllerRoute(pattern: "{controller=Home}/{action=Index}/{id?}")`
- Адрес = `/ИмяКонтроллера/ИмяМетода/параметр`
- `/` → `HomeController.Index()` (дефолтный контроллер + дефолтный action)
- `/Home/CreateBid` → `HomeController.CreateBid()`
- `/Credits/Edit/3` → `CreditsController.Edit(id: 3)`
- Ссылки: `<a asp-controller="Home" asp-action="CreateBid">` — tag helper

**Ключевое:** Осипов хочет услышать слова "система маршрутизации", "Routing", "формирование маршрутов", и чтобы ты **показал Program.cs** где это написано.

## Вопросы из записи защиты 28.03

### Лаба 1 (WPF/WinForms)

| Вопрос Осипова | Правильный ответ |
|----------------|-----------------|
| "Как у вас флажок блокирует кнопку? Какой механизм?" | **Команды (Commands)** — не события! В WPF есть механизм команд: `ICommand`, `CanExecute` определяет доступность кнопки |
| "Что такое триггеры?" | Триггеры в WPF — автоматическое изменение свойств при выполнении условия. Например: если `IsChecked=True` → `Background=Green` |
| "Покажите где в коде реализовано изменение цвета/статуса" | Нужно уметь быстро найти нужный файл и показать конкретный метод |
| "Сложно ли сделать цветовой перелив?" | Нет, просто указывается — через стили или триггеры |

### Лаба 2 (Razor Pages)

| Вопрос Осипова | Правильный ответ |
|----------------|-----------------|
| **"Каким образом формируется картинка? Откуда из источника?"** | `_Layout.cshtml` — мастер-страница (шапка, навигация). `@RenderBody()` подставляет содержимое конкретной страницы (Index.cshtml, Product.cshtml) |
| **"Покажите в проекте, в виде кода, где это реализовано"** | Открыть `Index.cshtml` (разметка) + `Index.cshtml.cs` (логика). Показать `_Layout.cshtml` → `@RenderBody()` |
| **"Как формируется маршрут?"** | `Program.cs` → `app.MapRazorPages()`. Имя страницы в папке Pages = адрес в URL. `/Product` → `Pages/Product.cshtml` |
| **"Где написано как формируется маршрут?"** | В файле `Program.cs` — строка `app.MapRazorPages()`. Это система маршрутизации (Routing) |
| "Напишите вместо Index другое название страницы" | Просто меняешь URL в браузере: `/Product` → откроется Product.cshtml |

### Лаба 3 (MVC)

| Вопрос Осипова | Правильный ответ |
|----------------|-----------------|
| **"Из каких частей состоит рутинг?"** | `{controller}/{action}/{id?}` — имя контроллера, имя метода действия, необязательный параметр |
| **"Как формируется ссылка? Что стоит за ссылкой?"** | `<a asp-controller="Home" asp-action="CreateBid">` — tag helper генерирует `/Home/CreateBid` |
| **"Как в MVC отдаётся вид?"** | Контроллер вызывает `return View()` — ищет файл `Views/{Controller}/{Action}.cshtml` |
| "Чем отличается от Razor Pages?" | В Razor Pages — файл страницы сам является endpoint. В MVC — контроллер отдельно, view отдельно |
