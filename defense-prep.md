## Лаба 1 — WinForms

### Реальные вопросы Осипова:

**1. Модальные и немодальные окна — в чём разница?**
- **Модальное** — блокирует главное окно, пока не закроешь. Пример: MessageBox, настройки
- **Немодальное** — не блокирует, можно переключаться между окнами. Пример: поиск
- В коде:
  - Модальное: `form.ShowDialog()` — возвращает `DialogResult`
  - Немодальное: `form.Show()` — не блокирует, код продолжает выполняться

**2. Калькулятор — на что смотрит:**
- Работоспособность (все кнопки работают)
- Форма не должна растягиваться: `FormBorderStyle = FixedSingle`, `MaximizeBox = false`
- Если обычное приложение — наоборот, форма должна быть "резиновой"

**3. Как сделать резиновую форму?**
- Свойство **Anchor** у элементов: Top, Left, Right, Bottom — элемент "привязывается" к краям формы
- Свойство **Dock** — элемент заполняет всю сторону (Top, Fill, Bottom и т.д.)
- Пример: `textBox1.Anchor = AnchorStyles.Top | AnchorStyles.Left | AnchorStyles.Right;`

**4. Покажите операцию вторичного окна (ЛР 6)**
- Графика, Chart, анимация — нужно уметь показать код Paint-события
- `Graphics g = e.Graphics;` в обработчике `Paint`
- `g.DrawLine(pen, x1, y1, x2, y2);` — рисование линий

### Что ещё может спросить:

| Тема | Ответ |
|------|-------|
| Что такое MDI? | Multiple Document Interface — одно родительское окно, много дочерних. `IsMdiContainer = true` |
| Как обработать нажатие клавиши? | `KeyPress` событие, `e.KeyChar` — символ, `e.Handled = true` — отмена |
| BackgroundWorker зачем? | Чтобы UI не зависал при долгих операциях. `DoWork` — фон, `ProgressChanged` — обновление UI |
| async/await vs BackgroundWorker | async/await — современный подход, проще код. BW — старый, но наглядный |
| Сериализация — как? | `System.Text.Json.JsonSerializer.Serialize(obj)` — в JSON, `Deserialize<T>(json)` — обратно |
| Что такое UserControl? | Составной элемент управления — комбинация нескольких контролов в одном переиспользуемом компоненте |

## Лаба 2 — ASP.NET Razor Pages

### Реальные вопросы Осипова:

**1. Цена с точкой или запятой — как клиент видит, как сервер?**
- **Клиент (браузер):** `<input type="number">` — принимает точку (`.`), зависит от локали браузера
- **Сервер (ASP.NET):** использует `CultureInfo` — в русской локали разделитель `,` (запятая)
- Проблема: клиент шлёт `31.5`, сервер ожидает `31,5`
- Решение: `decimal.TryParse(value, NumberStyles.Any, CultureInfo.InvariantCulture, out result)`
- Или настроить в `Program.cs`: `app.UseRequestLocalization("en-US")`

**2. Как формируется картинка (страница) в Razor Pages?**
- `_Layout.cshtml` — мастер-страница (шапка, навигация, футер)
- `@RenderBody()` — место куда вставляется содержимое конкретной страницы
- `_ViewStart.cshtml` — указывает Layout по умолчанию для всех страниц
- `_ViewImports.cshtml` — подключает пространства имён и Tag Helpers
- Конкретная страница (Index.cshtml) — подставляется в RenderBody

**3. Как формируется адрес? Маршрутизация в Razor Pages**
- `Program.cs` → `app.MapRazorPages()` — регистрирует все страницы из папки Pages
- Адрес формируется из **пути к файлу**: `Pages/Product.cshtml` → `/Product`
- `Pages/Movies/Index.cshtml` → `/Movies` или `/Movies/Index`
- Директива `@page` — обязательна, без неё страница не является endpoint
- `@page "{handler?}"` — добавляет параметр маршрута для handler'ов
- `@page "{id:int}"` — параметр id типа int

### Что ещё может спросить:

| Тема | Ответ |
|------|-------|
| Что такое PageModel? | Базовый класс для code-behind страницы Razor. Содержит OnGet/OnPost методы |
| OnGet vs OnPost | OnGet — при GET-запросе (загрузка страницы). OnPost — при POST (отправка формы) |
| Что такое Tag Helpers? | `asp-for`, `asp-page`, `asp-action` — серверные атрибуты, генерируют HTML |
| Что такое BindProperty? | Автоматическая привязка данных из формы к свойству модели |
| EF Core — что это? | ORM — объектно-реляционное отображение. Работаем с объектами C#, EF транслирует в SQL |
| Что такое миграция? | Изменение схемы БД на основе изменений в модели. `dotnet ef migrations add` |
| Что делает SeedData? | Заполняет БД начальными данными при первом запуске |

## Лаба 3 — ASP.NET MVC

### Реальные вопросы Осипова:

**3. Как формируется адрес в MVC?**
- `Program.cs` → `app.MapControllerRoute(pattern: "{controller=Home}/{action=Index}/{id?}")`
- Формат: `/ИмяКонтроллера/ИмяМетода/параметр`
- Примеры:
  - `/` → `HomeController.Index()`
  - `/Home/CreateBid` → `HomeController.CreateBid()`
  - `/Credits/Edit/3` → `CreditsController.Edit(3)`
- Контроллер = первый сегмент URL, Action = второй сегмент

**4. Из каких частей состоит контроллер как класс?**
- **Конструктор** — принимает зависимости через DI (DbContext, ILogger)
- **Методы действия (Action Methods)** — определяют что возвращаем:
  - `IActionResult` — базовый тип возврата
  - `View()` — возвращает представление (HTML)
  - `RedirectToAction()` — перенаправление
  - `Json()` — JSON ответ
  - `PartialView()` — частичное представление
  - `string` — просто текст
- **Атрибуты**: `[HttpGet]`, `[HttpPost]`, `[Authorize]`, `[ResponseCache]`
- **Приватные методы** — вспомогательные (например, GiveCredits())

**5. Маршрутизация: /контроллер/action метод**
- По умолчанию: `{controller=Home}/{action=Index}/{id?}`
- `Home` — дефолтный контроллер, `Index` — дефолтный action
- `id?` — необязательный параметр
- Суффикс "Controller" в имени класса **не пишется** в URL

### Что ещё может спросить:

| Тема | Ответ |
|------|-------|
| MVC — что означает? | Model-View-Controller. Model — данные, View — отображение, Controller — логика |
| Чем MVC отличается от Razor Pages? | MVC: контроллер отдельно от view. Razor Pages: code-behind привязан к странице. MVC лучше для API и сложных проектов |
| Что такое ViewBag? | Динамический объект для передачи данных из контроллера в представление |
| ViewBag vs Model? | ViewBag — нетипизированный (dynamic), Model — строго типизированный. Model предпочтительнее |
| Что такое [Authorize]? | Атрибут — ограничивает доступ только для авторизованных пользователей |
| Roles = "admin" что делает? | Доступ только для пользователей с ролью admin |
| Что такое AJAX? | Асинхронный запрос к серверу без перезагрузки страницы. Используем fetch() |
| Partial View — зачем? | Переиспользуемый фрагмент HTML. Подгружается через AJAX без перезагрузки |
| Identity — что это? | Библиотека ASP.NET для аутентификации/авторизации — регистрация, логин, роли |
| Code First — как работает? | Сначала пишем C# классы (модели), потом EF генерирует таблицы в БД через миграции |

5. **Ключевые файлы которые нужно знать:**
